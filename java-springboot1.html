什么是SpringBoot?

此处可参考SpringBoot官方网站,官方网站中为我们介绍了SpringBoot的主要功能如下
Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can "just run".
We take an opinionated view of the Spring platform and third-party libraries so you can get started with minimum fuss.
Most Spring Boot applications need minimal Spring configuration.

这段话的翻译如下
Spring Boot使创建独立的、生产级的基于Spring的应用程序变得很容易，您可以“直接运行”这些应用程序。
我们对Spring平台和第三方库有自己的看法，这样您就可以轻松入门了。
大多数Spring启动应用程序需要最少的Spring配置。
通过上述的文字我们可以总结出:
1.SpringBoot是Spring的产物.
2.SpringBoot可以使我们创建独立的,生产级的spring应用程序变得更容易,我们要做的="just run".
那么,SpringBoot到底是什么东西?
Spring Boot是由Pivotal团队提供的全新框架，其设计目的是用来简化新Spring应用的初始搭建以及开发过程。
该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置。
通过上述文字我们可以了解到,Spring使我们搭建Spring项目变的容易了很多,那么,容易在哪儿了呢?
首先,Springboot采用组合注解的方式,将Spring需要用到的配置,封装在@SpringbootApplciation注解中.
我们只需要创建一个类,给他加上@SpringBootApplication注解,即可达到我们SSM常规的配置文件的功能.
于是,我们免去了SSM整合的诸多配置文件,一个注解即可达到效果
对于这个注解为什么这么神奇的分析请看文章SpringBoot关键注解@SpringBootApplication分析
当然,SpringBoot不仅仅是帮我们省去了配置文件.他还帮我们处理了我们百分之九十以上的依赖版本.
我们在使用SpringBoot时,需要在pom.xml导入一个父类.

这个父类内部帮我们封装了多数我们需要用到的依赖的版本,让我们不必担心版本冲突引来不必要的麻烦
我们调用依赖的时候,只需要调用相应依赖的starter(启动器)即可
另外我们的Springboot的启动速度是非常快速的,我们的SMM框架使用的时候,每次Run都要等待很久
而我们的SpringBoot,短短几秒就可以达到启动的效果.
至于,为什么启动这么快呢.我[个人]的看法是,SpringBoot省去了许多的配置文件,
减少了编译所需的时间应该是其中的一个原因,至于其它的原因,还没有深究

到这里,你认识SpringBoot了么?让我用一句话概述他就是:
SpringBoot是利用组合注解@SpringBootApplicaiton并通过parent标签配合启动器实现依赖版本管理来简化Spring开发的一个优秀框架
